## 🏗️ 架构特点分析

### 优点：
1. **分层清晰**：从用户层到物理层，职责明确
2. **单例模式**：JLinkManager、SVDManager 等使用单例，确保资源唯一
3. **插件化设计**：通过 DevicePatchInterface 支持多厂商设备
4. **异步支持**：MCP 工具函数都是异步的
5. **类型安全**：使用 Pydantic 模型确保数据类型正确

### 问题：
1. **设备定义文件缺失**：`JLinkDevices.xml` 不存在，导致 Flagchip 补丁无法工作
2. **工具函数命名冲突**：需要使用别名（`_list_jlink_devices`）避免冲突
3. **GDBServer 与其他管理器职责不清晰**：GDBServer 应该是管理器而不是独立模块
4. **缺乏统一的错误处理**：各个工具函数的错误处理不一致
5. **插件发现机制弱**：只能手动注册 FlagchipPatch，无法自动发现
6. **资源路径硬编码**：tool/ 路径写死在代码中
7. **厂商内容耦合**：厂商特定内容（设备定义、SVD）与主体代码混在一起

## 📦 厂商包隔离方案

### 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **方案 A: Python 包** | 版本管理清晰、依赖独立、可发布到 PyPI | 需要维护多个仓库、用户需要单独安装 | 多厂商支持、正式产品 |
| **方案 B: 资源目录** | 简单直观、无需额外安装 | 无版本管理、路径管理复杂 | 个人使用、快速原型 |
| **方案 C: 配置文件** | 灵活可配置 | 配置复杂、用户体验差 | 高度定制化场景 |
| **方案 D: 混合模式** | 兼顾灵活性和易用性 | 实现复杂度中等 | 推荐方案 ⭐ |

### 推荐方案：混合模式（方案 D）

支持三种加载方式，按优先级顺序：

#### 1. 环境变量方式（优先级最高）
```bash
# 设置厂商资源目录
export JLINK_MCP_VENDOR_DIR=/path/to/vendor-resources

# 或者设置特定厂商
export JLINK_MCP_FLAGCHIP_DIR=/path/to/flagchip
export JLINK_MCP_STM32_DIR=/path/to/stm32
```

#### 2. Python 包方式（推荐）
```bash
# 安装核心包
pip install jlink-mcp

# 安装厂商包
pip install jlink-mcp-flagchip
pip install jlink-mcp-stm32
```

#### 3. 内置资源方式（默认）
```
jlink_mcp/
├── vendors/              # 厂商资源目录
│   ├── flagchip/
│   │   ├── devices.xml   # 设备定义
│   │   ├── svd/          # SVD 文件
│   │   └── flash/        # Flash 算法
│   ├── stm32/
│   └── nxp/
```

### Python 包入口点

```python
# jlink_mcp_flagchip/__init__.py
from jlink_mcp.vendor_interface import VendorPackage

class FlagchipVendor(VendorPackage):
    @property
    def name(self) -> str:
        return "flagchip"
    
    @property
    def version(self) -> str:
        return "1.0.0"
    
    def get_device_definitions(self) -> Path:
        return self.package_dir / "devices.xml"
    
    def get_svd_files(self) -> list[Path]:
        return list((self.package_dir / "svd").glob("*.svd"))
    
    def get_extensions(self) -> dict:
        return {
            "custom_tools": self.load_custom_tools()
        }
```
